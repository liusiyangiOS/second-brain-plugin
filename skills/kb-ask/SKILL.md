# kb-ask — 问你自己的第二大脑

从你**已经炼好的知识**里作答，而不是从公网现搜。因为笔记入库时已被炼成原子化半成品并双链，所以只需读「MOC + 少量相关笔记」就能又快又准地回答——这就是"不用 RAG"的底气。

## 一句话：kb-ask 和普通提问有什么不同

- 你问普通 AI：它答的是**公网/训练数据**里的通识，跟你没关系。
- 你问 kb-ask：它答的是**你自己沉淀过的东西**——你读过什么、当时怎么理解、留过什么疑问，并且能指回"这个结论在你哪条笔记里"。

所以它不是"再搜一遍"，而是"翻你自己的第二大脑"。库里没有，它就如实说没有，绝不拿公网内容冒充你的沉淀。

## 何时用

- 用户想从自己库里查/答："我之前记过啥""库里有没有关于 X""根据我的知识库回答""我对 X 是怎么理解的"。

## 前置：确认知识库位置（首次会问一次，之后不再问）

```bash
bash "${CLAUDE_PLUGIN_ROOT}/scripts/resolve_kb.sh"
```

- 打印出一行路径 → 作为知识库根（记为 `<KB>`）。
- 退出码 3（未配置）→ **只问一次**："请告诉我你的知识库根目录绝对路径（例如 `~/Documents/知识库`）"，然后：

```bash
bash "${CLAUDE_PLUGIN_ROOT}/scripts/set_kb.sh" "<用户给的绝对路径>"
```

## 流程

### 1. 检索候选（结构化，不做 RAG）

从问题里提关键词，跑共享检索原语：

```bash
bash "${CLAUDE_PLUGIN_ROOT}/scripts/kb_search.sh" <关键词1> <关键词2> ...
```

### 2. 由地图到细节

- **先读命中的领域 MOC**（`<KB>/_MOC/*.md`）——它是入口，能快速定位该主题下有哪些条目、有哪些开放问题。
- 再读命中靠前的**少量**原子笔记，优先看 `## TL;DR` 与 `## 核心论断`；需要细节或想看你当时的思考时，再看 `## 我的疑问与发散`。
- 不要一次读一大堆——原子笔记的意义就是让你只读必要的几条。

### 3. 作答并溯源

- 用中文基于笔记内容回答。
- **标注来源**：答案关键处指回具体笔记，如"（见 [[notes/2026-07-05-子代理编排]]）"，方便你顺藤摸瓜。
- 若不同笔记有冲突或演进，如实呈现"你先前记的是…，后来修正为…"。

### 4. 库里没有时

- 明确说"**知识库里暂时没有**关于 X 的条目"。
- 可以另行用公网知识补充，但**必须区分**哪些来自你的库、哪些是外部补充，别混为一谈。
- 顺带提示：如果这是个值得沉淀的问题，可用 `kb-digest` 消化后补一条进库。

## 原则

- **先地图后细节**，只读必要的几条笔记。
- **答案可溯源**到具体笔记。
- **库内 / 库外分清**，不拿公网内容冒充你的沉淀。
