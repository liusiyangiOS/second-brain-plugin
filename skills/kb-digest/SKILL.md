# kb-digest — 先消化，再入库

把外部材料变成"你真正读懂、并且连你的疑问都留下来"的知识。这是第二大脑里最日常的一条流。

## 它产出的「原子笔记」是什么

**原子笔记 = 一条只讲一个概念、能独立看懂、并连向相关条目的小笔记**，存放在知识库的 `notes/` 目录。它和你知识库里已有的那种"整篇技术文档"（放在 `技术分享/` 等分类目录里的成品）**不是一回事**：

- 整篇成品：一篇完整文章，信息量大，适合通读——那是 `kb-organize` 归档的东西。
- 原子笔记：把一篇材料**炼**成三段（一句话精华 + 核心结论 + 你的疑问），小、可连接、便于日后只读几条就答题。

原子笔记的灵魂是第三段 **「我的疑问与发散」**——把你当时没懂的点、追问、联想都留下。结论会过时，但"你当时怎么想的"是别人抄不走的复利资产。

## 何时用

- 用户发来链接 / 文章 / 视频 / 开源仓库 / 文件，想先搞懂再决定要不要沉淀。
- 用户说"这个是干啥的""帮我总结""我看看懂不懂"。
- 用户在一轮讲解后说"懂了 / 入库 / 收进去 / 沉淀下来"。

## 前置：确认知识库位置（首次会问一次，之后不再问）

```bash
bash "${CLAUDE_PLUGIN_ROOT}/scripts/resolve_kb.sh"
```

- 打印出一行路径 → 作为知识库根（记为 `<KB>`）。
- 退出码 3（未配置）→ **只问一次**："请告诉我你的知识库根目录绝对路径（例如 `~/Documents/知识库`）"，然后：

```bash
bash "${CLAUDE_PLUGIN_ROOT}/scripts/set_kb.sh" "<用户给的绝对路径>"
```

入库前先读 `<KB>/README.md`（原子笔记格式与目录约定的唯一事实源）。

## 流程

分四步，**不要跳过第 2、3 步直接入库**——先读懂是这条流的意义所在。

### 1. 取材

- 链接 → 用 `WebFetch` 抓取正文（视频链接抓简介/字幕页；仓库抓 README）。
- 文件 / 目录 → 用 `Read` 读。
- 粘贴文本 → 直接用。

抓不到就如实说明，别脑补内容。

### 2. 讲解 + 总结（先不入库）

用中文给出：

- **一句话定性**：这材料是干啥的、解决什么问题、适合谁。
- **结构化摘要**：核心观点 / 关键步骤 / 重要结论，分点列清。
- **和你已有知识的关系**（可选）：先用检索原语扫库里有没有相关旧条目，有就点一句"这和你之前记的 X 有关"：

  ```bash
  bash "${CLAUDE_PLUGIN_ROOT}/scripts/kb_search.sh" <关键词1> <关键词2> ...
  ```

讲完**停下来**让用户消化。明确告诉用户：可以继续追问，觉得没问题了说一声就入库。

### 3. 承接追问与发散（关键差异点）

- 用户不懂就解释，想展开就一起发散。
- **边答边记**：把这一轮关键问答整理成"疑问与发散"草稿（先记在脑子/草稿里，不落盘）。这些是复利的种子，别丢。
- 反复第 3 步，直到用户说读懂了 / 要入库。

### 4. 消化后入库 + 编织

用户示意入库后：

1. 按 `<KB>/README.md` 的原子笔记格式，在 `<KB>/notes/` 建 `YYYY-MM-DD-简短标题.md`：
   - `## TL;DR`：把材料炼成半成品（一到三句）。
   - `## 核心论断`：关键结论。
   - `## 我的疑问与发散`：**填入第 3 步沉淀的问答**——这一节是本技能的灵魂。
   - frontmatter 填 `source`（原链接/文件）、`tags`、`status: 已消化`。
2. **接着走 `kb-weave` 技能**：找相关旧条目、建双向链接、更新对应 MOC（复利引擎）。
3. 用 `commit_kb.sh` 提交推送**本次涉及的所有文件**（新笔记 + 被更新的旧笔记 + MOC），只提交这些、不 `git add -A`：

   ```bash
   bash "${CLAUDE_PLUGIN_ROOT}/scripts/commit_kb.sh" "docs(知识库): 消化并沉淀 <主题>" <file1> <file2> ...
   ```

## 原则

- **先读懂再沉淀**：没消化就别入库，否则只是搬运。
- **疑问必留**：用户当时不懂的点、追问、展开，和结论一样值钱。
- **不臆造**：沉淀的是材料本身 + 用户的真实思考，不自行编造结论。
